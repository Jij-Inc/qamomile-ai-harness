
The intended workflow is:

1. Prepare dataset summaries with `qcoder-prepare`.
2. Give an Agent only `benchmarks/cases`, `skills/qamomile_qcoder`, `prompts`, and the installed Python environment.
3. Ask the Agent to write Qamomile solution files without web access.
4. Evaluate with `qcoder-batch`.

QCoder's public repository evaluates by calling the QCoder web API with a session cookie. This project adds a local simulator-based evaluator for small cases so no-web agents can iterate without external access.

## Initial Benchmark Scope

The downloaded Hugging Face dataset contains 15,252 submissions across 67 problems. It contains submission histories, while the problem statements and web-API evaluation driver live in `references/qcoder-benchmark`.

The first local smoke set is:

- `QPC001_A1`: prepare `|+>` using one `h`.
- `QPC001_A2`: prepare uniform superposition, currently tested at `n=2`.
- `QPC001_A3`: prepare Bell state using `h` and `cx`.

These cases deliberately avoid the QCoder web API and evaluate Qamomile output locally by transpiling the Qamomile kernel to a Qiskit circuit, then checking:

- Python/Qamomile runtime errors.
- Unauthorized gates.
- Circuit depth.
- State-vector fidelity, ignoring global phase when the case says so.

Qamomile `transpile()` and `to_circuit()` treat top-level kernels as executable entrypoints and require classical I/O. State-preparation solutions naturally return quantum handles, so the evaluator uses Qamomile's build/emit pipeline for local circuit extraction instead of the public entrypoint shortcut.

By default, `qcoder-eval` and `qcoder-batch` run candidate files in an isolated Python subprocess with a timeout. This mirrors the strongest lesson from the neighboring harness in `../yu_workspace/projects/qamomile_ai_bench`: benchmark candidates should not run inside the long-lived orchestrator process. The in-process path remains available with `--in-process` for debugging.

See `docs/external_harness_learnings.md` for the concrete lessons imported from that harness and the larger schema features that are worth adding next.
